# Playwright Environment Fixes

## 1. Login Hydration
Updated `global-setup.cjs` to explicitly wait for the visibility of `#username-input`. Added `CI=1` to ensure non-interactive execution.
```javascript
await page.waitForSelector("#username-input", { state: "visible", timeout: 20000 });
```

## 2. Seed Data Hardening
Updated `backend/apps/projects/management/commands/seed_e2e_state.py` to:
- Assign `pw_owner`, `pw_reviewer`, and `pw_approver` to project indicators so they have visibility and action permissions.
- Seed essential `MasterValue` records (statuses, priorities, etc.) so the Admin masters edit UI has data to operate on.

## 3. Selector Stability
- Replaced fragile text selectors with more robust scoped locators (e.g., searching within the `main` or `project card` containers).
- Updated modal title expectation from `h2` to `h3` to match the actual component structure.
- Used numeric regex for extracting `projectId` from URLs to avoid capturing non-numeric path segments.

## 4. Authentication Stability
Ensured all tests utilize the storage states generated by `global-setup.cjs` (`pw_admin.json`, etc.) instead of repetitive manual logins, significantly improving test speed and reliability.
