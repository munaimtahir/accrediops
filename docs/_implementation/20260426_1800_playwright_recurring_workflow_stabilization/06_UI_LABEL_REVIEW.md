# UI Label Consistency Review

## Terminology Normalization
Conducted a review of user-facing labels to ensure they align with the simple accreditation tracker philosophy.

## Label Changes

| Old Label | New Label | Reason |
|-----------|-----------|--------|
| Project Home | Dashboard | Standardizes the entry point for project-level overview. |
| Admin Dashboard | Settings | Groups technical and administrative configurations under a common term. |
| Inspection Mode | Review / Inspection | Clarifies the dual purpose of the view for both internal review and audit inspection. |
| Export History | Reports | More intuitive for end-users looking for outputs. |
| Admin Frameworks | Framework Setup | More descriptive of the administrative task. |
| Import Logs | Import History | Standardizes "History" for log-based views. |
| AI Action Center | AI / Assist | Normalizes the AI section title in the detail page for consistency. |

## Unchanged Labels
- **Submit / Approve:** Preserved as standard workflow verbs.
- **Evidence / Gap:** Preserved as core accreditation concepts.
- **Worklist:** Preserved as the primary workbench term.

## Outcome
The UI now uses simpler, more direct language that focuses on the task at hand rather than the underlying technical structure.
