# AI instructions for developers

## Primary instruction

Do not build features that weaken the governance-centered, evidence-centered nature of the system.

## Build priorities

1. Protect source-of-truth integrity
2. Protect review/approval integrity
3. Preserve indicator-level operational clarity
4. Preserve recurring evidence workflows
5. Support reuse across clients safely
6. Support print-ready hard-evidence outcomes

## Specific anti-drift instructions

Do not reframe the app as:
- a general CRM
- a general DMS
- a customer-facing portal
- a lightweight compliance dashboard with no real operations
- an AI-first autopilot tool

Do not remove or weaken:
- recurring evidence tracking
- hard evidence preparation
- batch print / pack output
- evidence versioning
- internal review/approval
- clone/reuse logic for future clients

## When uncertain

When uncertain, prefer:
- explicit workflow
- explicit commands
- explicit history
- manual confirmation
- repeatable structure
- reusable templates
