# Recommended additions and deferrals

## Recommended additions

### 1. Physical evidence location tracking
Reason:
many accreditors still value hard evidence and physical inspection readiness.
The system should be able to record:
- binder name
- shelf/cabinet
- room
- physical file label

### 2. Print pack builder
Reason:
this is a major operational differentiator.
The system should prepare ordered print sets for inspection days.

### 3. Recurring evidence due dashboard
Reason:
recurring items are easy to miss and are central to real compliance.

### 4. Client variable replacement engine
Reason:
this is essential for reusing the same framework and documents for future clients.

### 5. Evidence reuse rules
Reason:
some evidence can be reused, some must be regenerated, and some require manual approval.
This should be explicit.

### 6. Framework analysis summary
Reason:
when a new accreditation checklist is loaded, the app should summarize the mix of indicator types and work burden.

## Recommended deferrals

### 1. Full automation with AI APIs
Not needed first.
Manual AI is safer and more aligned with your preference.

### 2. Complex notification engine
Useful later, but not core enough for the first build.

### 3. Fancy analytics dashboards
Only add after the workflow is trusted.

### 4. Client portal or client login
Out of scope and strategically unnecessary.

### 5. Billing or subscription functions
Not aligned with the business model.
