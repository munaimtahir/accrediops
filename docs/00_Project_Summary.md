# Project summary

## Product identity

This product is an **internal accreditation operations platform** used by your own service team to prepare, manage, monitor, and finalize accreditation work for clients.

It is **not** a public software product.
It is **not** a subscription platform.
It is **not** a client-facing portal.

The product exists to help your company deliver accreditation services with more structure, speed, reuse, and evidence discipline.

## Core purpose

The system should allow the team to:

1. load or define an accreditation checklist/framework
2. break it into indicators
3. classify indicators by evidence/work pattern
4. identify what each indicator actually requires
5. track the evidence and closure state for each client
6. maintain recurring evidence where needed
7. generate or organize documentation
8. review and approve readiness internally
9. export and print final evidence outputs for inspection use
10. reuse the same accreditation structure for future clients

## Strategic value

The real value of the product is not only tracking one client.
The real value is that once the team has built a complete accreditation structure for one framework, the same structure, templates, evidence logic, and document patterns can be reused for future clients with controlled client-specific replacement.

## Manual AI stance

AI should be used first in a manual/advisory mode:
- guidance
- drafting
- classification support
- readiness suggestions
- evidence assessment recommendations

AI should not control the application state.
