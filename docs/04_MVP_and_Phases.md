# MVP and phased rollout

## MVP principle

The MVP should focus on the minimum system that lets your team actually run accreditation work for real clients in a controlled way.

The MVP is not the smallest possible demo.
It is the smallest operational product that preserves the real service workflow.

---

## MVP scope

### 1. Framework and indicator core
- framework register
- checklist import
- domain / standard / indicator structure
- required evidence definition
- indicator classification
- recurring cadence classification

### 2. Client workspace core
- client register
- client-specific checklist instance
- assignment
- priority
- target date
- gap summary
- action required

### 3. Indicator operations core
- indicator list
- indicator detail workbench
- status tracking
- comments
- status history
- filters and search

### 4. Evidence core
- add/link/upload evidence
- evidence type
- evidence versioning
- current evidence marker
- evidence readiness tracking

### 5. Recurring evidence core
- mark indicators as recurring
- show daily/weekly/monthly due indicators on dashboard
- allow digital or scanned evidence update path
- blank recurring sheet print support

### 6. Review core
- request review
- return for revision
- approve
- final evidence ready
- submitted
- audit history

### 7. Output core
- Excel export
- filtered monitoring exports
- ordered print pack generation list
- batch print-ready output structure

### 8. AI core
- manual AI-guided drafting
- manual AI-guided evidence assessment
- AI recommendation saved with human review

---

## Phase 2

- template library
- client-variable replacement engine
- reusable document generation support
- framework cloning for new client
- recurring coverage calendar
- evidence pack builder refinement
- physical evidence location map
- stronger dashboards and analytics

---

## Phase 3

- richer AI-assisted classification
- larger reusable evidence intelligence layer
- multi-framework comparative views
- deeper recurring workflow automation
- optional advanced printing/pack assembly
- more advanced reporting

---

## Deferred / not needed now

- client portal
- subscriptions
- billing engine
- public marketing site integration
- autonomous AI decisioning
- over-engineered workflow automation
